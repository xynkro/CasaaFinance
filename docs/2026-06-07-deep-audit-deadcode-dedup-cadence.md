# FinancePWA — Deep Audit: Dead Code · Duplication · Output Cadence
> **Date:** 2026-06-07 · **Method:** 8-agent graphify-grounded audit, every candidate grep-verified against real usage (cron workflows, LaunchAgents, CLI entries, imports, JSX). · **Cost:** ~2.0M tokens / 312 tool-uses.

## Executive summary
- **596 LOC of confirmed-dead code** — verified by grep, far below the graph's raw "927 isolated / 896 uncalled" (which were inflated by schema fields, data JSON, pytest tests, and JSX components).
- **Verification mattered:** ~140 graph "dead" candidates were FALSE POSITIVES (16 in telegram.py via module-alias calls, 63 PWA components used via `<JSX/>`, etc.). Blindly trusting the graph would have broken imports.
- **A real latent BUG found** (not dead code): `scripts/signal_feedback.py:763` calls `send(report, topic=...)` but `send()` has no `topic` kwarg → `TypeError` swallowed by try/except → the signal-feedback report **silently never reaches Telegram**.
- **`vendor/skills/` is NOT dead weight** (my earlier hypothesis was wrong): ~18k LOC, but wired live via 2 cron workflows → wrapper scripts → per-skill subprocess. Only 1 standalone diagnostic CLI is dead.
- **Highest-value structural issue:** two independently-tuned CSP/CC scanners both write `scan_results` and will disagree on the same ticker/day.
- **Cadence:** the 3-brain cadence is slightly too much, concentrated in the twice-weekly WSR-lite. Recommended: cut the Friday WSR-lite.
